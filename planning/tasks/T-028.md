# T-028 Add ergonomic follow-up task generation

## Description

The spec's "Better Task Creation Ergonomics" asks for more ergonomic follow-up
generation, and the workflow rule states that follow-up work discovered during
verification should become new task files. `internal/taskrail/service.go`
already constructs a follow-up shape during verify (dependencies set to the
source task id, inheriting the parent's `spec_ref`).

Expose this as a first-class ergonomic on the scaffold command from T-027:
`taskrail task new --follow-up <id>` (or `taskrail task follow-up <id>`)
pre-fills `spec_ref` from the parent, adds the parent as a dependency, and seeds
the body noting the task derives from the parent's verification or discovery.

## Acceptance

- A follow-up flag or subcommand creates a new task that inherits the parent's
  `spec_ref` (overridable), lists the parent in `dependencies`, and records the
  provenance in the body.
- It reuses T-027 id allocation and validation, with the same `--json` support.
- Unit tests cover follow-up creation, `spec_ref` inheritance, and dependency
  wiring in a temporary repository.

## Verification Notes

- Run `go test ./internal/taskrail`.
- Manual test in a temp sandbox: create a parent task and a follow-up, run
  `validate` and confirm valid; persist `plan.md` and `report.md` under
  `planning/artifacts/manual-test/T-028/<timestamp>/`.

## Implementation Notes

- Build on T-027's `CreateTask`; factor the existing follow-up construction in
  `service.go` (near the verify follow-up shape) into the shared path so
  verify-time and manual follow-ups agree.
