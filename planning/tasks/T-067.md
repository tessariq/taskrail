# T-067 spec activate prints a one-line coverage summary after repointing the spec

## Description

After `taskrail spec activate <version>` (T-060) repoints the active spec, print
the one-line coverage summary for the newly-active spec, reusing the shared
coverage computation from `taskrail coverage` (T-059) — no duplicated logic.
This immediately answers "what is uncovered in the spec I just activated?" at
the moment of the switch, which is exactly when it matters (a freshly-activated
spec typically starts largely uncovered). The summary is informational only:
activation still succeeds regardless of the coverage figure, and `spec activate`
remains a repoint-only operation that never advances task status or fabricates
work. The line appears in the human view and as a field in `--json`. Spec:
`specs/v0.3.0.md#active-spec-activation`.

## Acceptance

- After a successful `spec activate <version>`, the command prints the one-line
  coverage summary for the now-active spec, computed via the shared T-059
  capability (no duplicated computation).
- Coverage is informational: activation succeeds and exits zero regardless of
  the percentage; it still never touches task status or task files.
- `--json` output includes the coverage summary field alongside the activation
  result.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, activate a spec with known uncovered areas; confirm the printed
  one-liner matches `taskrail coverage`'s summary and the `--json` field agrees.
