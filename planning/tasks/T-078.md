# T-078 WinGet release prerequisites: fork microsoft/winget-pkgs and add WINGET_TOKEN secret

## Description

One-time external setup that the automated WinGet publication (T-058) depends on
at release time but that is **not** self-controlled from CI or code. T-058 wires
the GoReleaser `winget` block and completes on a green `goreleaser check`, but the
live cross-repo pull request to `microsoft/winget-pkgs` needs infrastructure that
must exist before the first v0.3.0 tag: a fork of `microsoft/winget-pkgs` owned by
the project account/org, and a **classic** Personal Access Token with `public_repo`
scope stored as the `WINGET_TOKEN` repository secret (the default `GITHUB_TOKEN`
cannot open a cross-repository PR). This task tracks that ops setup so a tagged
release does not silently fail at the winget step. No code changes. Spec:
`specs/v0.3.0.md#windows-distribution-via-winget`.

## Acceptance

- A fork of `microsoft/winget-pkgs` exists under the project account/org and is
  the fork the T-058 `winget` block's `repository` points at.
- A classic PAT with `public_repo` scope is created and stored as the
  `WINGET_TOKEN` repository secret, wired to the GoReleaser release step.
- The chosen first-submission bootstrap path (manual `wingetcreate` first vs
  GoReleaser opening the first PR) matches what T-058 documents in
  `docs/workflow/releasing.md`; the release checklist reflects these prereqs.
- No source or task-state changes; this is release infrastructure only.

## Verification Notes

- Confirm the fork is reachable and the `WINGET_TOKEN` secret is present in repo
  settings; a snapshot `goreleaser release --snapshot` (from T-058) plus the
  configured token is the evidence that release-time publication can proceed. The
  live upstream PR and Microsoft moderation are a release-time outcome, not gated
  here (see T-058's completion boundary).

## Implementation Notes

- Ops/setup task, deliberately code-free. It gates the release through T-061's
  dependency list so the pre-release check cannot pass with the fork/secret
  missing; T-058 (the code + config) stays independent of it.
