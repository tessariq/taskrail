# T-101 taskrail-gap skill: structural gap --json plus agent semantic gap review

## Description

The structural gap signals (T-099) are the mechanical half; the semantic half —
edge cases, cleanup, rollout nuance the binary cannot infer — is delegated to a coding
agent through a skill, per `specs/v0.4.0.md#gap-analysis` and the LLM stance
(`specs/v0.4.0.md#recommendation-about-llm-support`). Ship a `taskrail-gap` skill that
runs the structural detector, then nudges the agent to reason semantically over the
result and propose candidate tasks — never auto-creating state.

## Acceptance

- New packaged skill `internal/taskrail/skills/taskrail-gap/SKILL.md` (embedded,
  installed by `init --with-skills`), invoking the binary via `${TASKRAIL:-taskrail}`.
- Documented flow: `coverage --gaps --json` (structural candidates) → agent reviews
  semantically for gaps the binary cannot detect → agent proposes candidate tasks →
  human promotes chosen ones via `task new` / `import --apply`. The skill never
  auto-creates tasks.
- The skill states the mechanical-vs-semantic split: structural signals come from the
  binary; the agent supplies judgement, and its output is candidates for human review,
  not committed state.
- `task skills:regen` run; committed `.agents/`/`.claude/` copies byte-identical
  (parity check green).
