# T-090 coverage --area: distinguish unknown vs sub-area vs deferred anchor in rejection error

## Description

Follow-up from T-077. `coverage --area <anchor>` rejects every non-coverable
anchor with one generic message — `--area %q is not a coverable area of %s`
(`internal/taskrail/coverage.go` `filterCoverageToArea`) — regardless of *why*
the anchor is not coverable. Three distinct cases collapse into the same text:

- an anchor that exists nowhere in the spec (unknown),
- a `####` sub-area that only rolls up into its `###` parent (valid heading, but
  not a coverable area on its own),
- a `###` area excluded by a `> Deferred to` / `> Subsumed by` marker.

Reported by go-reviewer on T-077 as a low-severity UX nit (non-blocking). This
task improves discoverability by tailoring the rejection message per case (e.g.
point a sub-area at its parent, note a deferred/subsumed area is intentionally
out of the denominator, and suggest `spec show --anchors` for an unknown one),
without changing the read-only, advisory contract or the exit code. Spec:
`specs/v0.3.0.md#deterministic-linkage-analysis`.

## Acceptance

- Rejecting a `--area` anchor distinguishes the three cases (unknown, `####`
  sub-area roll-up, deferred/subsumed area) with a message that names the case
  and points to the fix (parent area / intentional exclusion / `spec show
  --anchors`).
- A coverable-area anchor still filters exactly as before; the exit code for any
  rejection stays non-zero and no state/task file is written (read-only).
- The classification reuses the existing slug/area-parse logic (no re-slugging,
  no second spec parse beyond what coverage already does).
- `README.md`/`CHANGELOG.md` updated only if the user-facing wording change
  warrants it.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, run `coverage --area` for an unknown anchor, a `####` sub-area,
  and a deferred/subsumed area; assert each rejection names its case and the
  exit code stays non-zero with a clean tree.
