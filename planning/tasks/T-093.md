# T-093 coverage --area: reject empty and duplicate-slug ### anchors instead of matching by first-match scan

## Description

Follow-up from T-090 (go-reviewer, low/pre-existing). `coverage --area
<anchor>` matches the requested anchor against the parsed `###` area slugs by a
first-match linear scan (`internal/taskrail/coverage.go`, `CoverageForArea` and
`areaRejectionError`). Two degenerate specs slip through:

- A `### ` heading whose title slugs to the empty string (e.g. a punctuation-only
  title, via `slugHeading`) parses to `parsedArea{anchor: ""}`, so `--area ""`
  can *match* that bogus area and produce a hollow single-area report instead of
  a rejection.
- Two `###` headings that slug to the same anchor collapse: the first-match scan
  silently picks one and the other is unreachable via `--area`.

This is not a T-090 regression — the same first-match behavior predates it — but
the anchor-classification work made the boundary explicit. The fix keeps the
read-only, advisory contract and the non-zero rejection exit code; it only makes
`--area` reject an empty or ambiguous anchor rather than silently binding to a
degenerate/first-of-N area. Spec: `specs/v0.3.0.md#deterministic-linkage-analysis`.

## Acceptance

- `--area ""` (and any anchor that slugs to empty) is rejected with a clear
  message and a non-zero exit, never matched to an empty-slug `###` area.
- When two `###` areas slug to the same anchor, `--area <anchor>` is rejected as
  ambiguous (naming the collision) rather than silently binding to the first —
  or the ambiguity is surfaced as a `validate`/`coverage` diagnostic; decide and
  document which, keeping `coverage` read-only and advisory.
- A normal, unambiguous coverable anchor still filters exactly as before; no
  state or task file is written (read-only) and the exit code contract is
  unchanged.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, author a spec with a punctuation-only `###` title and a pair of
  same-slug `###` titles; assert `--area ""` and the duplicate anchor are both
  rejected with non-zero exit and a clean tree, while a normal area still scores.
