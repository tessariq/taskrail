# T-046 Add OS-specific CI builds (linux, windows, macos) with smoke and integration tests

## Description

Taskrail's CI currently builds and tests on a single OS, so OS-specific
regressions ship undetected. T-041's review surfaced a concrete example: the
pending-spec path comparison in `import --apply`
(`validateSpecRefWithPending`) compares an OS-native path from `parseSpecRef`
(`filepath.Clean`) against a slash-normalized `pending.path`
(`relPath` → `filepath.ToSlash`). On Windows those diverge, so every draft that
references its own pending spec would be wrongly rejected. The fix (apply
`filepath.ToSlash` before comparing) is committed but is **unverified on a real
Windows runner** — the darwin/Linux dev and CI environment cannot reproduce the
separator mismatch. More broadly, path handling, line endings, temp-dir
semantics, and file-mode assumptions are all places Taskrail can diverge per OS.

Add an OS matrix to CI (`.github/workflows/ci.yml`) that builds and tests on
Linux, Windows, and macOS, running the CLI smoke checks and the integration/behavior
tests on each so cross-platform regressions are caught before merge. Provision the
same pinned toolchain per OS via `jdx/mise-action` (mise.toml is the single source
of toolchain versions), consistent with the existing CI contract.

Follow-up derived from T-041's verification or discovery.

## Acceptance

- CI runs a matrix over `ubuntu-latest`, `windows-latest`, and `macos-latest`.
- Each OS job provisions the pinned toolchain (mise-action) and runs, at minimum:
  `go build ./cmd/taskrail`, `go vet ./...`, `go test ./...`, and the CLI smoke
  checks (`taskrail --help`, `taskrail validate`).
- Integration/behavior coverage that exercises real file/path flows (e.g.
  `import --apply` pending-spec resolution, spec-ref path comparison) runs on all
  three OSes, so the T-041 Windows path fix is actually exercised on Windows.
- Existing single-OS checks remain green; the matrix is authoritative (mirrors the
  current CI gate, does not weaken it).
- README/CI docs note the OS matrix if contributor-facing expectations change.

## Verification Notes

- Confirm the matrix by inspecting a CI run across all three OSes (link the run).
- Confirm the T-041 pending-spec path comparison passes on `windows-latest`.

## Implementation Notes

- Source: T-041 go-reviewer finding F1 (Windows path-separator mismatch), applied
