# T-023 Scrub existing dangling artifact references from committed files

## Description

Committed files already contain references to gitignored, machine-local
artifacts written by earlier verify runs. Examples include `planning/STATE.md`
(`last_verification_result`, `relevant_artifacts`) and task files such as
`T-017` (Implementation Notes referencing
`planning/artifacts/verify/T-017/.../report.json`).

Once the writers stop producing these references (T-021, T-022), perform a
one-time scrub of the dangling references already committed so no stale
local-only path remains in tracked files.

This task depends on T-021 and T-022 so the scrub is not immediately
reintroduced by a subsequent verify run.

## Acceptance

- No tracked file contains a path into a gitignored `planning/artifacts/`
  location after the scrub.
- `planning/STATE.md` retains a portable verification result consistent with the
  T-021 shape (no gitignored path).
- Task files retain their portable history; only the dangling artifact paths are
  removed or replaced with portable phrasing.
- `go run ./cmd/taskrail validate` passes after the scrub.

## Verification Notes

- Run a repository-wide search for `planning/artifacts/` references in tracked
  files and confirm none remain that point into gitignored content.
- Confirm `git status` is clean and `validate` reports `state valid`.

## Implementation Notes

- This is a content cleanup of tracked markdown, not a code change; keep the
  edits minimal and portable.
- Machine-managed `STATE.md` fields are normally CLI-owned; this scrub is a
  deliberate one-time correction tied to the portable-state contract, so editing
  them here is in scope.
