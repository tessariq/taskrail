# T-073 CI cross-compile smoke across windows/amd64, darwin, and linux/arm

## Description

Add a CI cross-compile smoke that builds `./cmd/taskrail` for the non-native
targets Taskrail ships to or intends to run on — `windows/amd64`,
`darwin/amd64`, `darwin/arm64`, and `linux/arm64` — on every push and pull
request, so a platform-specific compile break is caught immediately rather than
only at tag time when GoReleaser runs. This is a pure `GOOS`/`GOARCH` `go build`
matrix that publishes nothing; it complements T-058 (which adds the Windows
release build to GoReleaser) and conforms to the Taskfile/mise conventions
T-074 formalizes. Infra work gating the feature tasks. Spec:
`specs/v0.3.0.md#goals`.

## Acceptance

- CI builds `./cmd/taskrail` for `windows/amd64`, `darwin/amd64`, `darwin/arm64`,
  and `linux/arm64` on each push and pull request; a compile failure on any
  target fails CI.
- The smoke uses `go build` only (no publishing, no GoReleaser) and adds no
  runtime dependency.
- The build is invoked through a Taskfile target (consistent with T-074) and the
  toolchain is provisioned via mise.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- Run the cross-compile matrix locally (`GOOS`/`GOARCH` `go build`) for each
  target and confirm all succeed; confirm CI fails when a target is deliberately
  broken.
