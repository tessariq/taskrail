# T-022 Stop embedding local artifact paths in task notes

## Description

The verify writer also leaks a gitignored artifact path into committed task
files. In `internal/taskrail/service.go` the verify flow calls:

```
appendTaskNote(task, fmt.Sprintf("- %s: verification %s (%s)", nowText, input.Result, relReport))
```

`relReport` is a path under `planning/artifacts/verify/...`, which is
gitignored. Task files are committed, so cloning teammates see a note pointing
at a file that does not exist for them.

The appended note must record the portable result (timestamp and result, and
task context as needed) without a path into gitignored content.

## Acceptance

- The verify-time task note no longer contains a path into a gitignored
  artifact.
- The note still records the verification timestamp and result so the task
  history remains useful.
- Unit tests assert that no gitignored path is appended to task notes after a
  verify, using temporary directories.

## Verification Notes

- Run `go test ./internal/taskrail` and confirm the task-note tests pass.
- Manually verify a task in a temp sandbox and confirm its notes contain no
  `planning/artifacts/` path; persist artifacts under
  `planning/artifacts/manual-test/T-022/<timestamp>/`.

## Implementation Notes

- Keep the portable-result phrasing consistent with the state summary from
  T-021 so committed state and task notes describe results the same way.
