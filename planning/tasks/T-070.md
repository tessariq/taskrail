# T-070 status: add orphan/drift one-liner alongside the coverage summary

## Description

> Stretch: first to cut if v0.3.0 needs trimming.

Extend `taskrail status` (T-056) so its snapshot also reports a one-line
orphan/drift summary next to the existing one-line coverage summary, reusing the
same shared `taskrail coverage` computation (T-059) — no duplicated logic. The
coverage line answers "how much of the spec is covered?"; the orphan/drift line
answers "are any tasks pointing away from the active spec, and did any spec
areas gain no tasks?" at the same glance, without running a second command. It
stays strictly read-only (no STATE.md or task-file writes) and informational:
these remain advisory signals that never make `status` fail. The line appears in
the human view and as fields in `--json`. Spec:
`specs/v0.3.0.md#status-reporting`.

## Acceptance

- `taskrail status` prints a one-line orphan/drift summary (orphan-task count
  and uncovered-area count) alongside the coverage summary.
- `--json` includes the orphan and drift counts as stable fields.
- The summary reuses the shared T-059 coverage capability (no duplicated
  computation) and `status` stays read-only (clean working tree).
- The signals are advisory: `status` still exits zero and `validate` is
  unaffected when orphans/drift are present.
- `README.md` and `CHANGELOG.md` are updated for the added line.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox with one orphan task and one uncovered area, run `status` and
  `status --json`; assert the one-liner and JSON counts match `taskrail
  coverage` and the working tree stays clean.
