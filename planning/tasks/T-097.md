# T-097 Document the slug-in-id invariant and rename trap in task docs

## Description

The slug-in-id model is a trap operators learn only by hitting a `validate` failure.
Document it where they meet it, per `specs/v0.4.0.md#slug-in-id-invariant-documentation`.

## Acceptance

- Task docs state that the id and filename are two encodings of one identifier and
  that `validate` enforces `filename == "<id>.md"`.
- Docs cover the `task new` slug behavior (T-095) and the `task rename` flow (T-096).
- Docs call out the trap: a bare `git mv` to add a slug without changing the id yields
  `filename must be <id>.md`, and the correct fix is `task rename`.
