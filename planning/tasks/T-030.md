# T-030 Embed shippable skills and install on opt-in taskrail init --with-skills

## Description

Distribute the repo-agnostic skills from T-029 by embedding them in the binary
via `embed.FS` and writing them only on an explicit `taskrail init --with-skills`
opt-in. Per the spec and T-031, the default `taskrail init` must not silently
write agent-tool directories (`.agents/`, `.claude/`). This builds on the
version-aware `init` from T-019.

Taskrail ships skills as static, provider-agnostic text; it does not execute,
schedule, or orchestrate them.

## Acceptance

- The shippable skills are embedded in the binary via `embed.FS`.
- `taskrail init --with-skills` writes the embedded skills to the adopting repo
  (documented target directories); writing is opt-in only.
- The default `taskrail init` (no flag) writes no agent-tool or skill
  directories.
- Re-running with `--with-skills` is non-destructive (`writeFileIfMissing`
  semantics; no clobber of user-edited skills without confirmation), consistent
  with T-019.
- Unit tests cover: default init writes no skill directories; `--with-skills`
  writes the embedded skills; an idempotent re-run.
