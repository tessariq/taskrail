# T-051 Configurable skill entry point (${TASKRAIL:-taskrail}) with mise env override

## Description

Shipped skill text must call the Taskrail binary through an overridable
indirection, `${TASKRAIL:-taskrail}`, instead of a hardcoded `taskrail` or
`go run ./cmd/taskrail`. Adopting repositories need nothing: the default
resolves to the installed `taskrail` binary. This repository sets
`TASKRAIL="go run ./cmd/taskrail"` in `mise.toml` `[env]` (committed), so any
agent shell running under mise dogfoods the working tree rather than a stale
installed binary. This removes the reason the dogfooding and shipped skills
diverged and unblocks the repo adopting the packaged set (T-055). Spec:
`specs/v0.3.0.md#configurable-skill-entry-point`.

## Acceptance

- Every shipped skill under `internal/taskrail/skills/` invokes the binary as
  `${TASKRAIL:-taskrail} <args>`, with no hardcoded `taskrail` or
  `go run ./cmd/taskrail`.
- `mise.toml` `[env]` sets `TASKRAIL="go run ./cmd/taskrail"` (committed); a
  mise-run agent shell resolves the skills to the working-tree binary.
- With `TASKRAIL` unset, skill commands resolve to the installed `taskrail`
  binary.
- Any embedded-skill assertion covering the entry point passes.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- Exercise a shipped skill's commands in a sandbox with `TASKRAIL` set and
  unset; confirm the override resolves to `go run` and the default resolves to
  the installed binary.
