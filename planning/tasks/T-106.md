# T-106 status: active-spec drift breakdown

## Description

Improve `taskrail status` with a concise active-spec drift breakdown, per
`specs/v0.4.0.md#status-active-spec-drift-breakdown`. The output should answer
how much open work targets the active spec versus older/other specs without
changing `next` selection semantics.

## Acceptance

- Human `status` output includes a compact active-spec drift line with counts of
  open tasks linked to the active spec and open tasks linked away from it.
- When away-from-active open work exists, human output names each affected task
  id and its `spec_ref` in an inspectable section.
- `status --json` mirrors the breakdown with structured counts and task/spec-ref
  pairs.
- Only open work is counted: `todo`, `in_progress`, and `blocked` tasks count;
  `completed` and `cancelled` tasks remain delivered history and are excluded.
- The existing `next` warning remains unchanged and status stays strictly
  read-only, with no writes to `planning/STATE.md` or task files.
- README/help text documents the new status signal without describing it as a
  filter or scheduler.
