# T-066 Add taskrail coverage --min <pct>: opt-in CI exit code (validate stays advisory)

## Description

Add an opt-in `--min <pct>` flag to `taskrail coverage` (T-059) so a repository
can gate CI on spec coverage without touching `validate`. The spec keeps
coverage gaps strictly advisory — a gap must never make `validate` fail — so the
threshold lives only on the `coverage` command: when the computed coverage
percentage is below `--min`, `coverage` exits non-zero; otherwise it exits zero.
Absent `--min`, `coverage` always exits zero and behaves exactly as T-059. This
stays deterministic and provider-independent, consistent with the strategic
thread. Spec: `specs/v0.3.0.md#deterministic-linkage-analysis`.

## Acceptance

- `taskrail coverage --min <pct>` exits non-zero when coverage is below the
  threshold and zero when at or above it; the report output is unchanged.
- Without `--min`, `coverage` always exits zero (behavior identical to T-059).
- `validate` is unaffected: a coverage gap still never makes `validate` fail.
- `--min` composes with `--json` (the report is emitted; the exit code reflects
  the threshold).
- `README.md` and `CHANGELOG.md` document the `--min` CI-gating flag.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox with a known coverage percentage, run `coverage --min` above and
  below the actual value; assert the exit codes and that `validate` still passes.
