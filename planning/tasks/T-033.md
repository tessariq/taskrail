# T-033 Implement deterministic structural import

## Description

Implement the deterministic, no-LLM import path:
`taskrail import <src> --to tasks|spec|planning` parses markdown structure
(headings into spec sections, list items and subheadings into task drafts) into
draft files following the T-032 schema, reusing the T-027 scaffold for task
writes. The output is crude but real: the mechanical baseline and the `--apply`
ingest target for T-034.

## Acceptance

- `taskrail import <src> --to tasks` emits one task draft per detected unit
  (heading or bullet), validated against the T-032 schema and written via the
  T-027 scaffold (or as draft files for review).
- `--to spec` emits a spec draft; `--to planning` emits a combined bootstrap
  (spec draft plus task drafts plus a STATE seed).
- The command previews by default with an explicit apply, consistent with the
  project's dry-run discipline; no source file is modified.
- No LLM calls; output is deterministic for a given input.
- Unit tests over sample markdown cover each `--to` target using temporary
  directories.
