# T-057 Add snapshot-only taskrail stats command (--json)

## Description

Add `taskrail stats`: detailed aggregate statistics computed snapshot-only from
the current task files and STATE.md. Taskrail keeps no event log and
verification artifacts are ephemeral, so stats reports the current distribution,
not historical trends (time-series and any persistent event log are Explicitly
Excluded). Metrics: counts and percentages by status; blocked ratio and the
count of tasks with recorded blockers; spec-coverage percentage and a
per-spec-area breakdown; dependency shape (tasks with unmet dependencies,
longest dependency chain). The spec's previously-suggested "last-verification
tally" is out of scope — the spec's Statistics Reporting section now marks it so
(amended alongside this decomposition), because STATE.md keeps a single
path-free `last_verification_result` summary and Taskrail holds no event log, so
there is nothing to tally without introducing the persistent surface the spec
Explicitly Excludes. Read-only with `--json`. Reuses the coverage computation
from `taskrail coverage` (T-059) for the spec-coverage metric; it shares the
read-only reporting helper with `status` but does not depend on it (either
command may land first). Spec: `specs/v0.3.0.md#statistics-reporting`.

## Acceptance

- `taskrail stats` prints the aggregate metrics above; `--json` mirrors them.
- Stats is read-only (working tree stays clean) and computes from current files
  only — no new persistent store is introduced.
- Dependency-shape metrics (unmet dependencies, longest chain) are correct on a
  seeded DAG.
- The spec-coverage metric renders `N/A` when the active spec has no coverable
  areas (per T-059), not a false 0%/100%.
- Tests cover metric correctness and the read-only guarantee.
- `README.md` and `CHANGELOG.md` are updated for the new command.
- `AGENTS.md` documents `stats` as a read-only command (side-effect-free like
  `validate`), so agents know it needs no post-run `git status`/staging
  follow-up.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes
