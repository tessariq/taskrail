# T-012 Add taskrail version reporting

## Description

Add a way for the `taskrail` binary to report its own version. Today `taskrail --version`
fails with `unknown flag: --version`, which blocks tagging a real release. Provide both a
`taskrail version` subcommand and a `--version` flag, with the version injected at build time.

## Acceptance

- `taskrail version` prints the release version.
- `taskrail --version` prints the same version and exits without error.
- Version string is overridable at build time via `-ldflags` (no hardcoded constant as the only source).
- Default/dev build prints a sane fallback (e.g. `dev` or `0.0.0-dev`) when no version is injected.
- Output stays human-readable and minimal per the implementation bias.

## Verification Notes

- Add a smoke test asserting the subcommand and flag both emit a non-empty version line.
- Run `go run ./cmd/taskrail version` and `go run ./cmd/taskrail --version`.
- Document the `-ldflags` build invocation in README/Taskfile.

## Implementation Notes

- 2026-06-19: Added `cmd/taskrail/version.go` with build-time-overridable `var version` (default `0.0.0-dev`) and a `version` subcommand. Wired `Version` on the root command for `--version` with a minimal template. Added `cmd/taskrail/version_test.go` smoke tests. Documented `-ldflags` build in README and a `release` Taskfile target.
