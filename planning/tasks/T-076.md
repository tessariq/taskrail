# T-076 stats --format dot|mermaid: export the task dependency DAG

## Description

> Stretch: first to cut if v0.3.0 needs trimming.

Extend `taskrail stats` (T-057) with a `--format dot|mermaid` option that emits
the task dependency graph as Graphviz DOT or Mermaid text, so the decomposition
can be rendered and reviewed visually — the "aggregate insight into the task
graph" goal made inspectable. Nodes are tasks (labelled with id/title/status),
edges are `dependencies`. Strictly read-only and deterministic (stable node/edge
ordering), computed snapshot-only from the current task files; no new persistent
store and no rendering dependency (text output only, piped to the user's own
Graphviz/Mermaid tooling). Spec: `specs/v0.3.0.md#statistics-reporting`.

## Acceptance

- `taskrail stats --format dot` and `--format mermaid` emit a valid graph of the
  task DAG (nodes = tasks, edges = dependencies) with stable, deterministic
  ordering.
- The default `stats` output is unchanged when `--format` is omitted.
- The export is read-only (clean working tree), computes from current files
  only, and adds no runtime dependency (text only, no bundled renderer).
- Tests assert the emitted graph matches a seeded DAG for both formats.
- `README.md` and `CHANGELOG.md` are updated for the new option.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox with a known DAG, run both formats and confirm the graph text
  matches the dependencies and the working tree stays clean.
