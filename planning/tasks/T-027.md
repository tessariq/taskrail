# T-027 Add taskrail task new scaffold command

## Description

There is no task-creation command today; task files are hand-authored (how
`T-001`..`T-024` were produced). The spec's "Better Task Creation Ergonomics"
asks for task scaffolding, and "Agent Workflow Skills" wants skills to call a
real command instead of writing markdown by hand.

Add `taskrail task new` that writes a well-formed task file: allocate the next
sequential `T-NNN` id, set `title`, `status: todo`, `priority`, `spec_ref`,
`dependencies`, `updated_at`, and a template body (Description, Acceptance,
Verification Notes, Implementation Notes). Validate the result before it lands so
no invalid task is ever written. This command is the foundation skills (T-029)
and import (T-032/T-033/T-034) call instead of hand-authoring markdown.

## Acceptance

- `taskrail task new --title ... --spec-ref ... [--priority ...] [--dep T-0XX ...]`
  writes `planning/tasks/T-NNN.md` with the next free id and valid frontmatter
  plus a template body.
- Id allocation is deterministic (max existing id + 1, zero-padded) and
  collision-safe.
- The command refuses to write an invalid task (unknown spec anchor, nonexistent
  dependency) with a clear error; a `--json` output mode is supported for agents.
- `STATE.md` task counts stay consistent (todo incremented) by reusing existing
  state-count logic, not a hand-edit.
- Unit and smoke tests cover id allocation, validation rejection, and a
  successful scaffold in a temporary repository.

## Verification Notes
