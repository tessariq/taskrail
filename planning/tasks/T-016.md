# T-016 Upgrade release workflow actions off deprecated Node 20 runtime

## Description

The release workflow run for `v0.1.0` emitted a GitHub Actions annotation:

> Node.js 20 is deprecated. The following actions target Node.js 20 but are
> being forced to run on Node.js 24: goreleaser/goreleaser-action@v6.

This is currently cosmetic — runners force the action onto Node 24 — but the
warning will become a hard failure once Node 20 support is removed from the
runner image. Get ahead of it before it breaks `Release`.

## Acceptance

- The `Release` workflow runs with no Node.js deprecation annotations.
- `goreleaser/goreleaser-action` (and any other actions still on Node 20) are
  pinned to versions whose runtime targets a supported Node version.
- A tag push still produces a published GitHub Release with binaries and a
  non-empty body.

## Verification Notes

- Inspect the `Release` workflow run annotations after the change; confirm the
  Node deprecation warning is gone.
- Confirm `gh run view <id>` for the release job is green.
