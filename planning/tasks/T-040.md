# T-040 Consolidate import target validation behind a parseTarget helper

## Description

The T-034 go-reviewer flagged that import target validation is duplicated across
`Import`, `EmitImportPrompt`, and `ValidateImportDraft`: each independently checks
`validImportTargets`, and the `switch target` blocks add a third place to update
when a target is added. Introduce a small string-constant target type with a
canonical `parseTarget(string) (Target, error)` that the service methods call, so
the map lookup and the switch cases collapse to one source of truth.

Follow-up derived from T-034's verification or discovery.

## Acceptance

- A single `parseTarget` (or equivalent) is the one place that validates an import
  target; `Import` and `EmitImportPrompt` no longer re-check `validImportTargets`
  before delegating.
- Adding a new target requires editing one location; the `switch` becomes
  exhaustive or fails loudly on an unhandled target.
- No behavior change to existing import commands; tests still pass.

## Verification Notes

- `go test ./internal/taskrail` and `go run ./cmd/taskrail validate`.
