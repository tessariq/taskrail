# T-081 Ship autonomous-manual-test in the packaged skill set

## Description

Add `autonomous-manual-test` to the packaged (embedded) skill set so the shipped
set becomes a superset this repository can adopt without losing capability. This
is the manual-test half of the spec's *Ship autonomous-recovery And
autonomous-manual-test* section, split out from `autonomous-recovery` (T-054):
the recovery skill genuinely waits on the widened repair surface (T-072), but
`autonomous-manual-test` has no relationship to repair and is ready once the
entry-point indirection (T-051) lands â€” so it ships on T-051 alone rather than
being held behind the repair chain. Ship it as a repo-agnostic skill invoking
`${TASKRAIL:-taskrail}`, but keep its artifacts ephemeral and gitignored:
manual-test is NOT promoted to a product invariant â€” `init` must not provision
`planning/artifacts/manual-test/` and `validate` must stay unaware of it. This
resolves the v0.2.0 "Artifact And Init Consistency" deferral in favor of shipping
the skill without a new invariant. Record the manual-test ship-skill /
ephemeral-artifacts decision in the productization doc. Spec:
`specs/v0.3.0.md#ship-autonomous-recovery-and-autonomous-manual-test`.
