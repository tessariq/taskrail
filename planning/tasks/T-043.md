# T-043 Add an agent skill that drives the guided retrofit bootstrap

## Description

Retrofit is CLI-only today (T-035): no agent skill drives the guided bootstrap,
so an adopting coding agent has no scripted path for taking an existing repo
plus human notes to a Taskrail-managed layout. The shipped skill set
(`autonomous-backlog`/`task`/`verify`/`recovery`/`manual-test`) covers the
tracked-work loop but not onboarding/retrofit.

Add a repo-agnostic agent skill that encodes the guided retrofit flow: run
`taskrail retrofit <notes>` (dry run), review the proposed mapping and planning
bootstrap, apply on confirmation, then adopt the reviewed tasks (dependent on
T-042's persist path once it exists) and validate. The skill must invoke the
installed `taskrail` binary, be shippable via `init --with-skills`, and follow
the skills productization contract (`docs/workflow/skills-productization.md`)
with canonical + mirrored copies kept in sync.

Follow-up derived from T-035's verification or discovery.

## Acceptance

- A new repo-agnostic retrofit/onboarding skill exists under `skills/`, mirrored
  into `.agents/skills/` and `.claude/skills/`, and `./scripts/check-skill-mirrors.sh`
  passes.
- The skill drives detect -> dry-run -> confirm -> apply -> adopt -> validate
  using the installed `taskrail` binary (not `go run`), consistent with the
  other shippable skills.
- It is installed only opt-in via `taskrail init --with-skills`; a default init
  still writes no skill directories.
- Documented in the skills productization contract alongside the existing
  shipped skills.

## Verification Notes

- Run `./scripts/check-skill-mirrors.sh` and `go run ./cmd/taskrail validate`.
- Manual test: dry-run the skill against a temp non-standard repo and confirm it
  produces a valid Taskrail layout without clobbering existing files; persist
  `plan.md` and `report.md` under
  `planning/artifacts/manual-test/T-043/<timestamp>/`.
