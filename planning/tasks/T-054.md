# T-054 Ship autonomous-recovery in the packaged skill set

## Description

Add `autonomous-recovery` to the packaged (embedded) skill set so the shipped
set moves toward being a superset this repository can adopt without losing
capability. `autonomous-recovery` already routes every correction through
`taskrail repair` and forbids hand-editing authoritative state; shipping it needs
the entry-point indirection (T-051), since the skill already drives recovery
through `repair` as it stands today. The widened repair surface (T-072) is a
prerequisite: the spec's "Ship autonomous-recovery" section calls for the
Extended State Repair work before the skill ships, so `autonomous-recovery` lands
only after T-072 widens repair and shrinks the skill's human-resolved residue to
what the shipped skill claims. The sibling `autonomous-manual-test` skill from
the same spec section ships separately in T-081 (it has no repair dependency and
must not be held behind T-072). Update the productization doc disposition
(recovery: dogfooding-only -> shippable). Spec:
`specs/v0.3.0.md#ship-autonomous-recovery-and-autonomous-manual-test`.
