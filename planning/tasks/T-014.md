# T-014 Release check v0.1.0

## Description

Final readiness gate before tagging Taskrail `v0.1.0`. Confirm every v0.1.0 goal is met,
all spec commands behave as documented, the working tree is clean, and the release metadata
(CHANGELOG, version, tag) is in place. This task gathers the evidence for cutting the release;
it does not add new product behavior.

## Acceptance

- All v0.1.0 commands verified end-to-end against the spec: init, validate, next, start, complete, block, verify (incl. follow-up creation).
- `gofmt -l .` is clean, `go vet ./...` is clean, `go test ./... -race` passes.
- `taskrail version` / `--version` report `v0.1.0` from a release build (depends on T-012).
- Coverage is at least 80% (depends on T-013).
- `CHANGELOG.md` has a dated `## v0.1.0` section replacing `Unreleased`, listing the delivered commands.
- README and Taskfile reflect the release version and build invocation.
- A `git tag v0.1.0` is prepared (or the exact tagging command is documented for the maintainer).
- `planning/STATE.md` is consistent and `taskrail validate` reports a valid repo.

## Verification Notes

- Run `go run ./cmd/taskrail validate` and capture output.
- Record the release checklist results under `planning/artifacts/verify/T-014/`.
- Tagging itself is a maintainer action; this task confirms readiness and stages the metadata.
