# T-104 task new --area: active-spec spec_ref shorthand

## Description

Add `taskrail task new --area <anchor>` as an active-spec shorthand for
`--spec-ref <active_spec_path>#<anchor>`, per
`specs/v0.4.0.md#active-spec-task-authoring-shorthand`. The command should keep
all existing `task new` behavior, but let authors target an active-spec area
without copying the spec path by hand.

## Acceptance

- `task new --area <anchor>` resolves the created task's `spec_ref` to
  `STATE.md`'s active spec path plus the supplied anchor.
- `--area` and `--spec-ref` are mutually exclusive; passing both exits with a
  clear error and writes nothing.
- The anchor is validated against the active spec's real heading anchors using
  the same rules as `spec show <active-version> --anchors` / `validate`; unknown
  anchors fail before a task file or `STATE.md` is written.
- `--follow-up` keeps inheriting the parent `spec_ref` by default, and
  `--area` explicitly overrides that inherited ref when supplied.
- `--json` keeps the existing created-task shape and reports the resolved
  `spec_ref`.
- README/help/completion guidance covers the shorthand and points users at
  `spec show <active-version> --anchors` for valid anchors.
