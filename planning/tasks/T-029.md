# T-029 Produce a repo-agnostic shippable skill set

## Description

The current skills under `skills/` (mirrored into `.agents/skills/` and
`.claude/skills/`) are dogfooding scaffolding: they invoke
`go run ./cmd/taskrail`, which only works inside the Taskrail source tree, and
they hand-author task files. Per the decisions recorded in T-031, produce a
repo-agnostic shippable skill set that invokes the installed `taskrail` binary
and calls `taskrail task new` (T-027) instead of writing markdown by hand.

This repository's own dogfooding skills may keep `go run` until the binary is
the dogfooding entry point; only the shippable set must be portable.

## Acceptance

- A clearly separated shippable skill set exists, invoking `taskrail ...` (the
  installed binary) and never `go run`.
- Shippable skills call `taskrail task new` for task creation rather than
  hand-authoring task files.
- The repo-local dogfooding skills remain valid (may keep `go run`); the
  shippable-vs-dogfooding split matches the list recorded in T-031.
- `./scripts/check-skill-mirrors.sh` still passes.

## Verification Notes

- Run `./scripts/check-skill-mirrors.sh` and confirm it passes.
- Grep the shippable skills for `go run` and confirm none remain.

## Implementation Notes

- Keep canonical `skills/` as the source of truth; the shippable set is the
  input embedded by T-030.
- Coordinate with T-027 so the commands and flags the skills reference exist.
