# T-100 coverage --gaps --fail-on: opt-in gate on structural gap signals

## Description

Give the structural gap signals (T-099) an opt-in CI gate, mirroring the existing
`coverage --min` gate (`cmd/taskrail/coverage.go`). By default gap analysis stays
advisory and never fails; a `--fail-on` selector lets a repo choose to red the build on
specific gap categories, per `specs/v0.4.0.md#gap-analysis`.

## Acceptance

- `coverage --gaps --fail-on <category>` exits non-zero when any signal of the named
  category is present; categories match the T-099 signals
  (`missing-verification`, `dependency-anomaly`, `under-decomposed-area`), repeatable
  or comma-separated. Without `--fail-on`, `--gaps` exits zero regardless of findings.
- The gate lives only on the command's exit code (like `--min`); it never writes state
  and never makes `validate` fail — gap signals stay advisory to `validate`.
- `--json` output is unchanged by the gate; the gate only affects the exit code.
- A run with no coverable areas (or no matching signals) exits zero.
