# T-065 Add the shared taskrail spec parent command (empty parent for the spec family)

## Description

Land the `spec` parent cobra command as a single, shared surface so the
subcommands in the `spec` family (`activate` T-060, `list`/`show` T-062,
`add` T-063) attach to one parent instead of three tasks independently
introducing it and colliding on the same file. This task adds only the parent:
`taskrail spec` with no side effects, printing help/usage and listing its
subcommands as they are wired in. It writes nothing and reads nothing beyond
help text. Spec: `specs/v0.3.0.md#spec-inspection-and-authoring-commands`.

## Acceptance

- `taskrail spec` exists as a parent command that prints usage and its
  registered subcommands; invoked with no subcommand it is read-only and writes
  nothing.
- The parent is defined once in a single place that T-060, T-062, and T-063
  attach their subcommands to (no duplicate parent definition).
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, run `taskrail spec` and `taskrail spec --help`; confirm usage
  output, a clean working tree, and no state writes.
