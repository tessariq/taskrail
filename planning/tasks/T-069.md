# T-069 coverage reverse-map: show which task(s) cover each spec area

## Description

> Stretch: first to cut if v0.3.0 needs trimming.

Extend `taskrail coverage` (T-059) with a reverse map: for each coverable spec
area, list the task id(s) that cover it (directly or via the roll-up rule), and
for covered areas make the linkage auditable rather than reporting only the
aggregate percentage. This turns the coverage output from "how much is covered"
into "what covers what", which makes the percentage inspectable, surfaces
areas covered by an unexpected task, and flags double-coverage (several tasks
pointing at one area) worth reviewing. Strictly read-only and additive to the
existing report; reuse the shared coverage computation, do not re-slug or
duplicate logic. Both the human and `--json` views gain the per-area task list.
Spec: `specs/v0.3.0.md#deterministic-linkage-analysis`.

## Acceptance

- `taskrail coverage` lists, per coverable area, the covering task id(s)
  (respecting the roll-up rule), and marks areas with no covering task.
- `--json` includes the per-area covering-task list in a stable shape.
- Double-covered areas (more than one covering task) are distinguishable in the
  output.
- The reverse map reuses the shared coverage capability (no duplicated
  slug/anchor logic) and stays read-only (clean working tree).
- `README.md` and `CHANGELOG.md` are updated for the extended output.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox with a known spec/task set (one area covered by two tasks, one
  covered via a `####` child), run `coverage` and `coverage --json`; assert the
  per-area task lists, the double-coverage marking, and a clean tree.
