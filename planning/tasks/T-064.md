# T-064 Ship a packaged spec-command skill for coding agents (--with-skills)

## Description

Ship a repo-agnostic, packaged skill that guides coding agents through the
`spec` command family so they author tracked work against real spec anchors.
The skill invokes `${TASKRAIL:-taskrail}` (entry point from T-051), installs
opt-in via `taskrail init --with-skills`, and is mirrored into `.agents/skills/`
and `.claude/skills/`. Its flow: discover anchors with `spec show --anchors
--json` (the structured feed from T-068) before authoring a task, inspect specs
with `spec list`, advance the active spec with `spec activate`, and scaffold a
new spec with `spec add`. Because the JSON anchor feed (T-068) lands first, the
skill is authored once against it rather than being written against the human
listing and rewritten later. It stays static,
provider-agnostic skill text — Taskrail does not execute or orchestrate it —
consistent with the productization contract. Depends on the commands it
documents (T-060 activate, T-062 list/show, T-063 add), the JSON anchor feed it
consumes (T-068), the entry point (T-051), and the packaged-skills parity model
(T-055), so the skill ships under the package-parity check rather than the
retired three-way mirror check. Spec:
`specs/v0.3.0.md#spec-inspection-and-authoring-commands`.

## Acceptance

- A `spec`-command skill exists under `internal/taskrail/skills/`, invokes
  `${TASKRAIL:-taskrail}`, and installs via `init --with-skills`.
- The skill covers `spec show --anchors --json` (anchor discovery before
  `task new`), `spec list`, `spec activate`, and `spec add`, with no hardcoded
  binary path.
- It is mirrored per the packaged-skills convention and the package-parity
  check (T-055) passes with the skill included.
- Shippable-skills tests cover the added skill.
- `CHANGELOG.md`: the added spec-command skill folds into the single shared
  skills entry (with T-054), not its own line. Create that shared entry if it is
  absent and append to it if present, so the two tasks landing in either order
  never produce a duplicate line.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- `init --with-skills` in a sandbox lists the spec-command skill; follow its
  flow (`spec show --anchors` -> `task new --spec-ref`) and confirm the authored
  task validates.
