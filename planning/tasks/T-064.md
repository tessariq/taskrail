# T-064 Ship a packaged spec-command skill for coding agents (--with-skills)

## Description

Ship a repo-agnostic, packaged skill that guides coding agents through the
`spec` command family so they author tracked work against real spec anchors.
The skill invokes `${TASKRAIL:-taskrail}` (entry point from T-051), installs
opt-in via `taskrail init --with-skills`, and is mirrored into `.agents/skills/`
and `.claude/skills/`. Its flow: discover anchors with `spec show --anchors`
before authoring a task, inspect specs with `spec list`, advance the active spec
with `spec activate`, and scaffold a new spec with `spec add`. It stays static,
provider-agnostic skill text — Taskrail does not execute or orchestrate it —
consistent with the productization contract. Depends on the commands it
documents (T-060 activate, T-062 list/show, T-063 add) and the entry point
(T-051). Spec: `specs/v0.3.0.md#spec-inspection-and-authoring-commands`.

## Acceptance

- A `spec`-command skill exists under `internal/taskrail/skills/`, invokes
  `${TASKRAIL:-taskrail}`, and installs via `init --with-skills`.
- The skill covers `spec show --anchors` (anchor discovery before `task new`),
  `spec list`, `spec activate`, and `spec add`, with no hardcoded binary path.
- It is mirrored per the packaged-skills convention and the relevant
  skill/parity check passes (aligned with T-055's package-parity model).
- Shippable-skills tests cover the added skill.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- `init --with-skills` in a sandbox lists the spec-command skill; follow its
  flow (`spec show --anchors` -> `task new --spec-ref`) and confirm the authored
  task validates.
