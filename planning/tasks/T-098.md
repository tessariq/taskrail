# T-098 taskrail-decompose skill: draft spec-anchored tasks for uncovered areas via import --apply

## Description

Spec-to-task decomposition, per `specs/v0.4.0.md#spec-to-task-decomposition-support`,
turns spec headings and requirements into reviewable draft tasks anchored to real
`spec_ref` anchors. The mechanical parts already exist (`coverage --json` reports
uncovered active-spec areas; `spec show <version> --anchors` lists live heading
anchors; `import --apply` scaffolds real task files from a JSON `ImportDraft`). The
judgement-heavy part is left to a prompt/skill flow per the LLM stance. Ship this as a
**skill only** — no new binary surface — that composes those shipped primitives.

## Acceptance

- New packaged skill `internal/taskrail/skills/taskrail-decompose/SKILL.md` (embedded,
  installed by `init --with-skills`), invoking the binary via `${TASKRAIL:-taskrail}`.
- Documented flow: discover uncovered areas (`coverage --json`) and their live anchors
  (`spec show <version> --anchors`) → agent authors an `ImportDraft` whose every
  `spec_ref` targets a real anchor → `import --apply <draft>` → `validate`.
- The skill states its boundary versus the existing skills: `taskrail-decompose` is
  spec-driven (existing spec headings), `taskrail-import` is arbitrary-source-driven,
  `taskrail-retrofit` is whole-repo bootstrap — so the three do not read as duplicates.
- Draft-only invariant holds: the discovery/authoring steps write no committed state;
  `import --apply` (human-invoked, reviewed) remains the single writer.
- `task skills:regen` run; committed `.agents/`/`.claude/` copies are byte-identical
  (parity check `task check:skills` green).
