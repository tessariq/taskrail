# T-088 Make write-side filesystem errors path-portable (strip absolute repo path)

## Description

Follow-up from T-062's error-portability cleanup. The read-side sweep (T-062)
made every `os.ReadFile`/`os.ReadDir` error name a repo-relative path via the new
`fsCause` helper (`internal/taskrail/state_io.go`), but the **write-side**
filesystem errors still leak the user's absolute repository path — either via the
`%s` slot (an absolute `path` argument) or via the wrapped raw `*fs.PathError`
tail (`%w`).

Known offenders:

- `internal/taskrail/state_io.go`: `write state file`, `write task file`,
  `create directory %s` (ensureDir), `stat %s` / `write %s` (writeFileIfMissing).
- `internal/taskrail/skills.go`: `write backup %s`, `write %s` (both branches).
- `internal/taskrail/paths.go`: `create %s`, `write layout marker`.
- `internal/taskrail/import_apply.go`: `create %s` (taskDraftLabel path),
  `write imported spec`.
- `internal/taskrail/transitions.go`: `write verification plan`, `write
  verification report`, `write verification markdown report`.

`ensureDir` and `writeFileIfMissing` are package-level helpers with no repo root
in scope, so de-leaking them requires either threading a root/label parameter or
promoting them to methods — a small signature change kept out of T-062 to keep
that change focused on reads.

## Acceptance

- Every write-side filesystem error names a repo-relative path and never embeds
  the absolute repository root (mirror T-062's `relPath(...) + fsCause(...)`
  pattern; extend `fsCause` reuse rather than re-implementing).
- `ensureDir`/`writeFileIfMissing` no longer surface an absolute path (thread a
  root/label or convert to methods).
- Table/focused tests assert no-leak for a representative write failure per site
  class, and that `errors.Is` classification is preserved.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- Trigger a representative write failure (e.g. write target under a read-only or
  directory-occupied path) and assert the error text omits the temp repo root.
