# T-055 This repo adopts packaged skills; retire 3-way mirror for a package-parity check

## Description

This repository stops maintaining a bespoke `skills/` source and consumes the
packaged set materialized by `taskrail init --with-skills`, like any adopter.
Retire the custom `skills/` directory and the three-way
`scripts/check-skill-mirrors.sh` diff. Keep `.claude/skills/` and
`.agents/skills/` committed (zero-setup clone) but guard them with a single
package-parity check asserting the committed copies equal the embedded
`--with-skills` output, wired into CI and lefthook in place of the old mirror
check. Update `AGENTS.md`, `docs/workflow/skills-productization.md`, and
`docs/workflow/skills-overview.md` to describe one packaged set with a
configurable entry point, replacing the dogfooding-vs-shipped split. Depends on
the entry point (T-051), the force-reinstall/parity plumbing (T-053), and both
skills being shipped (T-054). Spec:
`specs/v0.3.0.md#this-repository-adopts-the-packaged-skills`.

## Acceptance

- The custom `skills/` source and `scripts/check-skill-mirrors.sh` are removed;
  `.claude/skills/` and `.agents/skills/` are regenerated from the package and
  committed.
- Every reference to the retired `check-skill-mirrors.sh` is swept, not just the
  skills docs: `AGENTS.md` (Source-Of-Truth Files, Workflow Checks, and Change
  Checklist mentions), `lefthook.yml`, and `.github/workflows/ci.yml`.
- A package-parity check asserts the committed copies equal `init --with-skills`
  output and runs in CI and lefthook in place of the old three-way check.
- `AGENTS.md` and the skills docs describe one packaged set with a configurable
  entry point; no dogfooding-vs-shipped split remains.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass,
  including the new parity check.

## Verification Notes

- Run the parity check green; introduce a deliberate drift and confirm it fails;
  confirm CI and lefthook invoke it in place of the retired mirror check.
