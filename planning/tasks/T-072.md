# T-072 Implement widened repair cases from the T-052 audit

## Description

Implement the additional mechanical repair cases enumerated by the T-052 audit,
widening `taskrail repair` beyond the v0.2.0 set. Each implemented case must be
one the audit proved deterministic: a single safe correction that neither
advances a status nor fabricates work. Repair still only ever rewrites STATE.md
(never a task file), defaults to a dry run showing the diff, and re-runs
validation after `--apply`. The hard boundary from the audit stays intact — a
missing `spec_ref`, a dependency cycle, and more than one in_progress task remain
human-resolved and are reported through validation, never auto-fixed. Spec:
`specs/v0.3.0.md#extended-state-repair`.

## Acceptance

- Repair fixes each mechanical case the T-052 audit qualified, with a single
  provably-safe correction; dry run shows the diff and `--apply` re-runs
  validation.
- Repair never rewrites a task file, never advances a status, never fabricates
  work.
- A missing `spec_ref`, a dependency cycle, and more than one in_progress task
  remain unrepaired and are reported through validation.
- Tests cover each new mechanical case and each untouched boundary case.
- `CHANGELOG.md` notes the widened `repair` surface.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- Seed sandboxes with each new mechanical drift and each boundary case; run
  `repair` (dry run then `--apply`) and record diffs plus re-validation output.
