# T-032 Define the import contract and draft schema

## Description

Define the `taskrail import` contract before implementing it. Specify the source
kinds (notes markdown, rough feature docs, bug notes, todo docs, draft specs)
and outputs (spec drafts, task drafts, combined planning bootstrap), the
`--to tasks|spec|planning` surface, and a stable structured-draft schema (JSON)
that the agent-driven path (T-034) emits and `--apply` ingests.

The binary performs no LLM calls; the semantic lift is the agent's
responsibility. Drafts map onto the T-027 task schema so ingest reuses scaffold
and validation rather than a divergent path.

## Acceptance

- A documented import contract: source kinds, `--to` targets, and the output
  draft shapes.
- A versioned JSON draft schema for tasks and spec sections that round-trips
  through `--emit-prompt` / `--apply` (T-034) and maps cleanly onto T-027's task
  fields (title, spec_ref, dependencies, body).
- The contract states the binary makes no LLM calls; semantic lift is the
  agent's responsibility, and Taskrail provides the docs/prompts.
- Schema validation rules are defined (required fields, id/dependency/spec_ref
  handling) reusing T-027 validation.
- No command behavior yet: this task delivers the contract and schema that
  T-033/T-034 implement against. Unit tests cover schema (de)serialization and
  validation of a sample draft.
