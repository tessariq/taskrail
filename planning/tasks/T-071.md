# T-071 shell completion for spec versions and anchors (feeds task new --spec-ref)

## Description

> Stretch: first to cut if v0.3.0 needs trimming.

Add shell-completion support for spec versions and spec anchors so authoring a
task with `taskrail task new --spec-ref <path>#<anchor>` is completable instead
of hand-typed. Completion draws on the `spec` family (T-062): version arguments
(for `spec show`/`spec activate`) complete to the versioned specs under
`specs/`, and `--spec-ref` completes to real `<path>#<anchor>` values using the
same anchor logic `spec_ref` validation accepts (reuse T-062's anchor source, do
not re-slug). This complements `spec show --anchors` (T-062) and the structured
anchor feed (T-068) with interactive discovery at the shell. Build on cobra's
existing completion machinery; no new runtime dependency. Read-only — completion
never writes STATE.md or task files. Spec:
`specs/v0.3.0.md#spec-inspection-and-authoring-commands`.

## Acceptance

- Version arguments (`spec show`, `spec activate`) complete to the versioned
  specs under `specs/`.
- `task new --spec-ref` completes to valid `<path>#<anchor>` values whose
  anchors match what `spec_ref` validation accepts (reuses the T-062 anchor
  source, no re-slugging).
- Completion is wired through cobra's completion command with no new runtime
  dependency and no state writes.
- `README.md` documents enabling completion; the `CHANGELOG.md` note folds into
  the single shared `spec` commands entry (with T-060/T-062/T-063), not its own
  line.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, exercise the completion functions for a spec version and a
  `--spec-ref` value; confirm the suggested anchors let an authored task pass
  `validate`.
