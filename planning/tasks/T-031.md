# T-031 Record the skills productization decisions

## Description

The spec requires three skills decisions be made explicitly: portability,
distribution mechanism, and the relationship to task-creation ergonomics. Record
the decisions (already taken) as a short contract that the implementation tasks
follow:

- **Portability.** Shipped skills must be repo-agnostic and invoke the installed
  `taskrail` binary, not `go run ./cmd/taskrail`.
- **Distribution.** One initial path: `embed.FS` skills written only on opt-in
  `taskrail init --with-skills`. Default `taskrail init` must not write
  agent-tool directories.
- **Relationship to task creation.** Shipped skills call the real
  `taskrail task new` command (T-027) instead of hand-authoring task markdown.

Taskrail ships skills as static, provider-agnostic text and does not execute,
schedule, or orchestrate them.

## Acceptance

- A short decision document (for example `docs/workflow/skills-productization.md`
  or a spec-adjacent note) records all three decisions with rationale and the
  non-goals (no skill execution, scheduling, or orchestration runtime).
- The document states the default `taskrail init` must not write `.agents/` or
  `.claude/` skill directories.
- The document names which repo-local skills are shippable versus
  dogfooding-only, as the input list for T-029.
- No code change is required; this task is the contract that T-029, T-030, and
  T-034 implement against.
