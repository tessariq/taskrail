# T-105 coverage --gaps --area: scoped structural gap review

## Description

Make structural gap analysis compose with active-spec area scoping, per
`specs/v0.4.0.md#gap-analysis`. After T-099 introduces `coverage --gaps`, this
task lets `coverage --gaps --area <anchor>` narrow the gap report to one
coverable active-spec area using the existing `coverage --area` vocabulary.

## Acceptance

- `coverage --gaps --area <anchor>` reports only gap signals for the named
  coverable active-spec area.
- Area resolution and rejection reuse the normal `coverage --area` behavior:
  unknown anchors, sub-area anchors, deferred/subsumed areas, empty slugs, and
  duplicate slugs produce the same class of clear errors and write nothing.
- `--json` mirrors the narrowed gap report and still identifies the active spec
  path and selected area.
- If no gap signals exist for the selected area, the command exits zero and
  reports an empty narrowed result.
- If T-100's `--fail-on` gate is also present, the gate evaluates the narrowed
  gap result rather than the full-spec result.
- The command remains strictly read-only and never changes `planning/STATE.md`
  or task files.
