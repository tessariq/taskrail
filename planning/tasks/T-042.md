# T-042 Let retrofit apply persist reviewed tasks via the import pipeline

## Description

Close the bootstrap-not-persisted asymmetry in `taskrail retrofit` (T-035).
Today `retrofit --apply` scaffolds the layout (`specs/`, `planning/tasks/`,
`STATE.md`, marker) but never persists the notes-derived task/spec drafts: they
are surfaced for review only, so `planning/tasks/` is empty after apply even
though the output reports N task drafts. This is deliberate — the structural
bootstrap is crude and non-authoritative, and `CreateTask` requires a `spec_ref`
that planning-import drafts lack — but it leaves a manual gap between "retrofit
applied" and "tracked work exists".

Wire retrofit into the existing review pipeline so reviewed tasks can actually
land: `retrofit --emit-prompt` should print the same agent prompt as
`import <notes> --to planning --emit-prompt`, and an agent-refined draft should
flow through `import --apply` to create real tasks. The retrofit command becomes
the single guided entry point for detect -> scaffold -> import -> adopt, rather
than stopping at scaffold.

Follow-up derived from T-035's verification or discovery.

## Acceptance

- `taskrail retrofit <notes> --emit-prompt` prints a ready-to-paste agent prompt
  equivalent to the planning-target `import --emit-prompt`, without scaffolding.
- A documented path takes an agent-refined draft from a retrofit bootstrap
  through `import --apply` so real spec/task files land, each task carrying a
  valid `spec_ref` (no invented anchors; reuse `CreateTask` validation).
- Retrofit's plain `--apply` behavior is unchanged (scaffold + marker + validate,
  no unreviewed tasks written); the persist path is explicit and opt-in.
- Unit tests cover the emit-prompt wiring and the end-to-end retrofit ->
  emit-prompt -> apply flow in a temporary directory.

## Verification Notes

- Run `go test ./internal/taskrail ./cmd/taskrail`.
- Manual test in a temp sandbox: retrofit a non-standard repo with notes, emit
  the prompt, hand-author a small draft, apply it, and confirm real tasks land
  and `validate` passes; persist `plan.md` and `report.md` under
  `planning/artifacts/manual-test/T-042/<timestamp>/`.

## Implementation Notes

- Compose existing surfaces (T-033/T-034 import + emit-prompt + apply); do not
  fork a second import path from retrofit.
- Keep the crude structural draft non-authoritative: real tasks land only after
  the review/apply step, never from the raw structural parse.
