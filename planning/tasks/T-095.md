# T-095 Slugged task creation: derive slugged id and filename from --title, with --slug override

## Description

`taskrail task new --title "…"` currently writes a bare `T-<n>` id and `T-<n>.md`
filename that ignore the title, producing opaque tasks amid slugged, human-scannable
siblings. Derive a slug from the title and write a slugged id with a matching
filename, per `specs/v0.4.0.md#slugged-task-creation`.

## Acceptance

- `task new --title "X"` writes id `T-<n>-<slug>` and filename `T-<n>-<slug>.md`
  where `<slug>` is `slugify(title)` (lowercased, non-alphanumeric runs collapsed to
  single hyphens, leading/trailing hyphens trimmed); `validate` passes with no edits.
- `task new --title "X" --slug short` uses `short` (itself normalized) instead of the
  derived slug; `validate` passes.
- `task new` with neither `--title` nor `--slug` keeps the bare `T-<n>` id and
  `T-<n>.md` filename; `validate` passes.
- The written id always equals the filename stem (`filename == "<id>.md"` holds).
- `--json` emits the created id and path.
