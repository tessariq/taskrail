# T-050 Conservative state repair surface (taskrail repair / validate --fix)

## Description

v0.2.0 spec State Repair (specs/v0.2.0.md#state-repair, soft 'should consider') deferred at v0.2.0 release. Add a conservative repair surface: proposes minimal safe correction per detected violation, defaults to dry-run showing diff before applying, only repairs mechanical inconsistencies (stale task counts, STATE.md field disagreeing with task files), never advances status or invents work, re-runs validation after applying. Removes autonomous-recovery skill's need to hand-edit authoritative state. Must respect Explicitly-Excluded guard: no status-advancing or work-fabricating repair.

## Skills Integration

The repair surface must be usable by coding agents through the skill layer, not
just the raw CLI. Design the command and its skill together so an agent drives
repair through Taskrail commands instead of hand-editing authoritative state.

- **Retarget the recovery skill.** Update `skills/autonomous-recovery/SKILL.md`
  (and its `.agents/skills/` and `.claude/skills/` mirrors) so the required flow
  routes through `taskrail repair` (dry-run, review diff, apply, re-validate) and
  drops the "bootstrap-era manual edits" fallback. The recovery skill must no
  longer instruct agents to bypass the CLI.
- **Ship a repair skill.** Provide a shippable, repo-agnostic skill that invokes
  the installed `taskrail` binary (not `go run ./cmd/taskrail`), consistent with
  the other shippable skills under `internal/taskrail/skills/`. Wire it into the
  `embed.FS` distribution so `taskrail init --with-skills` installs it; default
  `init` must stay silent. Mirror it into `.agents/skills/` and `.claude/skills/`
  and keep `./scripts/check-skill-mirrors.sh` green.
- **Deterministic contract.** The skill drives a deterministic dry-run → review →
  apply → re-validate loop and stops; it never advances status or fabricates
  work, matching the command's own guard and the Explicitly-Excluded runtime
  boundary (skills stay static provider-agnostic text; Taskrail does not execute
  or orchestrate them).

## Acceptance

- `taskrail repair` (or `validate --fix`) exists with the conservative dry-run →
  apply → re-validate contract; no status-advancing or work-fabricating repair.
- `skills/autonomous-recovery/SKILL.md` and both mirrors route recovery through
  `taskrail repair` and no longer permit hand-editing authoritative state.
- A shippable repair skill exists, invokes the installed `taskrail` binary, is
  distributed opt-in via `taskrail init --with-skills` (`embed.FS`), and is
  mirrored; `./scripts/check-skill-mirrors.sh` passes.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.
