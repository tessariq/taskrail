# T-034 Add agent-driven import via emit-prompt and apply

## Description

Add the agent-in-the-loop import path so coding agents do the semantic lift
while the binary stays LLM-free. `taskrail import <src> --emit-prompt` prints or
writes a ready-to-paste prompt plus the T-032 draft schema; the agent returns
structured drafts; `taskrail import --apply <draft.json>` validates and writes
real spec and task files via the T-027 scaffold. Ship the import prompt as a
repo-agnostic skill on the embed.FS path (T-030).

The thin `--llm` adapter (binary calling a model directly) is explicitly
deferred to v0.3 per the spec's preferred ordering.

## Acceptance

- `taskrail import <src> --emit-prompt` outputs a deterministic prompt embedding
  the source, the T-032 schema, and Taskrail conventions; no LLM call is made in
  the binary.
- `taskrail import --apply <draft.json>` validates the structured draft against
  T-032 and writes spec/task files via the T-027 scaffold; invalid drafts are
  rejected with clear errors.
- An import skill/prompt is shipped via the T-030 embed.FS opt-in path and
  invokes the installed `taskrail` binary.
- The thin `--llm` adapter is explicitly not implemented (deferred to v0.3) and
  documented as out of scope.
- Unit tests cover emit-prompt determinism and apply validation/round-trip using
  temporary directories.
