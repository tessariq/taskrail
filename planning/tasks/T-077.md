# T-077 coverage --area <anchor>: filter coverage/reverse-map to one spec area

## Description

> Stretch: first to cut if v0.3.0 needs trimming.

Add an opt-in `--area <anchor>` flag to `taskrail coverage` (T-059) that narrows
the report — coverage status, orphan/drift signals, and the reverse map (T-069)
— to a single coverable spec area, for focused "is *this* feature decomposed?"
checks without scanning the full report. The anchor is resolved with the same
slug logic `spec_ref` validation and coverage use (no re-slugging); an anchor
that is not a coverable area is rejected with a clear error. Read-only and
advisory like the base command; composes with `--json`. Spec:
`specs/v0.3.0.md#deterministic-linkage-analysis`.

## Acceptance

- `taskrail coverage --area <anchor>` restricts the report to the named coverable
  area (coverage status, orphan/drift, and the T-069 reverse map for that area).
- The anchor reuses the shared slug/anchor logic (no re-slugging); a non-coverable
  or unknown anchor is rejected with a clear error and no write.
- `--area` composes with `--json` and stays read-only (clean working tree); the
  signals remain advisory (`validate` unaffected).
- `README.md` and `CHANGELOG.md` are updated for the new flag.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, run `coverage --area` for a covered and an uncovered anchor and
  an invalid one; assert the filtered report, the error, and a clean tree.
