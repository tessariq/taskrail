# T-053 Force-reinstall skills on upgrade: init --with-skills --force with backup

## Description

`taskrail init --with-skills` currently skips when skills are already installed,
so upgrading the binary never refreshes materialized skill files. Add a
`--force` flag that reinstalls the embedded skills over existing copies. Before
overwriting a locally-modified skill file, back it up to a **timestamped** name
(so a second `--force` run never clobbers an earlier backup) and report what was
overwritten and what was backed up. Without `--force`, behavior is unchanged:
already installed skills are left in place. Spec:
`specs/v0.3.0.md#force-reinstall-for-skill-upgrades`.

## Acceptance

- `init --with-skills --force` overwrites installed skill files from the
  embedded set and reports the overwritten and backed-up paths.
- A locally-modified skill file is backed up to a timestamped name before
  overwrite and is recoverable; two successive `--force` runs produce distinct
  backups (the first is never clobbered).
- Without `--force`, an already-initialized repo is left unchanged (no writes).
- `--json` output stays consistent with the existing init path-reporting rules.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox: `init --with-skills`, modify a skill, re-run with and without
  `--force`; confirm skip vs overwrite-plus-backup, then force a second time and
  confirm both timestamped backups survive.
