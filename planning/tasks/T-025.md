# T-025 Reconcile init artifact scaffolding and manual-test convention with the gitignored-on-demand contract

## Description

`taskrail init` currently `ensureDir`s `planning/artifacts/verify/` and
`planning/artifacts/runs/` but not `planning/artifacts/manual-test/`, even
though the manual-test workflow writes there. After T-024, `validate` no longer
requires gitignored artifact directories to exist. This task resolves the
remaining half of the "Artifact And Init Consistency" decision so there is one
coherent artifact contract.

The decision: artifacts under `planning/artifacts/` are gitignored, reproducible
local output created on demand by `taskrail verify`; `init` should not pre-create
gitignored artifact output directories that a clean checkout drops anyway.
Manual-test artifacts remain an internal dogfooding convention and are not
promoted to a product invariant; `init` does not provision them.

## Acceptance

- Decision recorded explicitly: manual-test artifacts stay internal dogfooding,
  not a product invariant; `init` does not create `planning/artifacts/manual-test/`.
- `init` no longer pre-creates gitignored artifact output directories that a
  clean checkout would drop (verify creates `planning/artifacts/verify/<id>/<ts>/`
  on demand per T-024); if any artifact dir is still created, the reason is
  documented and it must not be required by `validate`.
- The artifact contract is documented in one place (README or `docs/workflow/`)
  consistent with T-021 and T-024: the artifacts tree is gitignored reproducible
  output, and neither committed state nor validation depends on it surviving a
  Git round-trip.
- Unit tests assert `init` does not create `planning/artifacts/manual-test/` and
  that a fresh `init` plus clean checkout still passes `validate` (consistent
  with the T-024 invariant).

## Verification Notes

- Run `go test ./internal/taskrail`.
- Manual test in a temp sandbox: `init`, inspect the created tree, confirm no
  manual-test directory, confirm `validate` passes; persist `plan.md` and
  `report.md` under `planning/artifacts/manual-test/T-025/<timestamp>/`.

## Implementation Notes

- `init` directory creation lives in `internal/taskrail/service.go` (`Init`,
  `ensureDir` / `writeFileIfMissing`); artifact directory fields live on the
  paths struct in `internal/taskrail/paths.go`.
- Keep aligned with T-024 (validate) and T-021 (portable state) so the artifact
  contract reads coherently across init, validate, and verify.
