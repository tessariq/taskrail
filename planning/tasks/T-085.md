# T-085 Harden task-ID allocation and detect numeric-prefix collisions

## Description

Two related defects, reproduced against v0.2.0 in a repo whose tasks are named
`T-NNN-slug.md` with frontmatter `id: T-NNN-slug` (report: `docs/taskrail_bug.md`).
This repo dogfoods bare `T-NNN` IDs, so it does not hit the bug, but the
codebase supports slug-suffixed IDs (validation only requires
`filename == id + ".md"`), and both code paths are fragile regardless of
convention.

**1. `nextTaskID` restarts at `T-001`.** `internal/taskrail/tasks.go:73` parses
the next number via `strconv.Atoi(strings.TrimPrefix(id, "T-"))`. For a
slug-suffixed id (`T-076-ingestion-commands`) that becomes
`Atoi("076-ingestion-commands")`, which errors and is silently skipped. When
every task is slug-suffixed, every id is skipped, `max` stays `0`, and the
generator returns `T-001` — colliding with the existing `T-001-*` task on its
numeric prefix. It also scaffolds a bare `nextID+".md"`
(`internal/taskrail/transitions.go:333`, `:437`) instead of the repo's
`T-NNN-slug.md` shape.

**2. `validate` misses numeric-prefix collisions.** `validateTasks`
(`internal/taskrail/validation.go:264`) dedups on the full id string, so
`T-001` and `T-001-milestone-v0.1.0` are treated as distinct and the collision
passes as `state valid`. Numeric prefix is the real identity (deps use it;
`taskIDPattern = ^T-\d+$` in `import.go:63`; maintainers renumbered colliding
prefixes in `8be87fc`), so two task files sharing `^T-(\d+)` must be an error.

## Acceptance

- `nextTaskID` derives the next number from the numeric prefix (`^T-(\d+)`) of
  every task id (and/or matching filenames), not exact-string `Atoi`; in a repo
  whose highest id is `T-102`, the next id is `T-103`, whether ids are bare or
  slug-suffixed.
- Bare-`T-NNN`-id repos (this one) keep current behavior: next after `T-084`
  stays `T-085`, filenames stay `T-NNN.md`, existing tests still pass.
- `validate` reports a duplicate-id error when two task files share the same
  `^T-(\d+)` numeric prefix (e.g. `T-001` and `T-001-milestone-v0.1.0`), instead
  of `state valid`.
- Table-driven unit tests cover: slug-suffixed max-prefix allocation, mixed
  bare/slug ids, and prefix-collision detection in `validate`. TDD: start each
  with a failing test.
- YAML colon-quoting in scaffolded frontmatter is preserved (see report note).
- Decide and document (in this task or a spec follow-up) whether scaffolding
  should emit `T-<next>-<slug>.md` to match a slug convention, or stay bare;
  keep the numeric-prefix invariant either way.
