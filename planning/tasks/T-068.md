# T-068 spec show --anchors --json: structured anchor feed consumed by the spec skill

## Description

Make `spec show <version> --anchors --json` (T-062) emit a stable, structured
list of the spec's live heading anchors, so an agent can pick a real anchor for
`taskrail task new --spec-ref <path>#<anchor>` from machine-clean output rather
than scraping the human anchor listing or reverse-engineering the slug rule. The
anchor set must be exactly what `spec_ref` validation accepts (same slug logic
reused by T-062). This feed lands before the packaged spec-command skill (T-064),
which consumes it directly — so the skill is authored once against the JSON
rather than being written against the human listing and rewritten here. Spec:
`specs/v0.3.0.md#spec-inspection-and-authoring-commands`.

## Acceptance

- `spec show <version> --anchors --json` emits a stable structured anchor list
  identical to the anchors `spec_ref` validation accepts.
- A task authored from a JSON-listed anchor passes `validate`.
- The stable JSON shape is documented so T-064's skill can consume it directly
  (this task produces the feed; the skill authored in T-064 is its consumer).
- `CHANGELOG.md` notes the `--anchors --json` output.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, read anchors via `spec show --anchors --json`, author a task
  with one of them, and confirm `validate` passes.
