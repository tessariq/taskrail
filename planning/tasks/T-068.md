# T-068 spec show --anchors --json: structured anchor feed consumed by the spec skill

## Description

Make `spec show <version> --anchors --json` (T-062) emit a stable, structured
list of the spec's live heading anchors, and have the packaged spec-command
skill (T-064) instruct agents to consume that JSON directly when authoring a
task, rather than scraping the human anchor listing. This closes the loop from
anchor discovery to `taskrail task new --spec-ref <path>#<anchor>` with
machine-clean output: the agent reads the JSON anchor list and picks a real
anchor without reverse-engineering the slug rule. The anchor set must be exactly
what `spec_ref` validation accepts (same slug logic reused by T-062). Spec:
`specs/v0.3.0.md#spec-inspection-and-authoring-commands`.

## Acceptance

- `spec show <version> --anchors --json` emits a stable structured anchor list
  identical to the anchors `spec_ref` validation accepts.
- The packaged spec-command skill (T-064) documents consuming `--anchors --json`
  as the anchor-discovery step before `task new --spec-ref`.
- A task authored from a JSON-listed anchor passes `validate`.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- In a sandbox, read anchors via `spec show --anchors --json`, author a task
  with one of them, and confirm `validate` passes.
