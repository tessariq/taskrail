# T-103 Warn when next selects non-active-spec work

## Description

`taskrail next` can currently select eligible work from a non-active spec with no
visible signal. Add the advisory warning described in
`specs/v0.4.0.md#active-spec-selection-warnings` without changing deterministic
selection order.

## Acceptance

- `next` keeps selecting by status, dependencies, priority, and stable id.
- If the selected `todo` task points outside `active_spec_path`, human output prints
  a warning naming the task, its `spec_ref`, and the active spec path.
- If the already-active `in_progress` task points outside `active_spec_path`, the
  same warning is attached to the selection result.
- `next --json` includes the advisory warning in a structured `warnings` field.
- `status` surfaces the same computed warning while remaining read-only.
