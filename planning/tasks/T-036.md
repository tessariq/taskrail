# T-036 Stop test fixtures from eagerly creating gitignored artifact directories

## Description

T-024 made `taskrail validate` stop requiring the gitignored
`planning/artifacts` and `planning/artifacts/verify` directories to exist, so a
clean checkout with no artifacts tree is valid and `taskrail verify` creates the
tree on demand.

The test fixtures have not been reconciled with that changed behavior. In
`internal/taskrail/service_test.go`, `seedFixtureRepo` still unconditionally
runs `os.MkdirAll(planning/artifacts/verify)`, recreating the very directory the
change deems non-required. This is harmless today, but it leaves the fixtures
asserting the pre-T-024 layout: a new test built on `seedFixtureRepo` silently
starts with the artifacts tree present, which can mask a regression in
`verify`'s on-demand creation logic.

Surfaced by the T-024 go-reviewer pass:
> `seedFixtureRepo` eagerly creates `planning/artifacts/verify`
> (`internal/taskrail/service_test.go`) ... "this is not incorrect ... but it
> creates maintenance friction ... contrary to the intent expressed by the
> change."

## Acceptance

- `seedFixtureRepo` no longer creates `planning/artifacts/verify` (or any
  artifacts output directory); fixtures represent a clean checkout where the
  gitignored artifacts tree is absent.
- Any test that genuinely needs a pre-existing artifacts tree creates it
  locally and explicitly, rather than relying on the shared fixture.
- No placeholder files (`.gitkeep`) are introduced.
- The full package suite still passes: `go test ./internal/taskrail -race`.
- `verify` on-demand creation coverage remains intact (the
  `TestVerifyCreatesArtifactDirsWhenAbsent` guard stays meaningful and passes).

## Verification Notes

- Run `go test ./... -race` and confirm green.
- Confirm no fixture or helper recreates `planning/artifacts*` implicitly:
  inspect `seedFixtureRepo` and any helpers it calls.

## Implementation Notes

- The fixture is `seedFixtureRepo` in `internal/taskrail/service_test.go`; only
  the `os.MkdirAll(.../artifacts/verify)` call needs removing, plus any test
  that depended on its side effect.
- Independent of product code; this is test-hygiene alignment with T-024.
