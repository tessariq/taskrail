# T-056 Add read-only taskrail status command (human overview + --json)

## Description

Add `taskrail status`: a strictly read-only overview of current tracked-work
state. It must never write STATE.md or task files — unlike `next`, which dirties
the tree by persisting `next_action`/`updated_at` even on a selection probe. It
may compute the next eligible task with the same selection logic but must not
persist it. The default human view reports: active spec version and path; task
counts by status (done / active / blocked / todo); the next eligible task marked
as not persisted; blocked tasks with recorded reasons; the last verification
result (the path-free summary already in STATE.md); and a one-line coverage
summary computed via the shared `taskrail coverage` capability (T-059). `--json`
mirrors the same fields for agent consumption. Spec:
`specs/v0.3.0.md#status-reporting`.

## Acceptance

- `taskrail status` prints the human overview and leaves the working tree clean
  (no STATE.md or task-file writes), asserted by a no-diff check.
- `taskrail status --json` emits the same fields in a machine-readable shape.
- The next eligible task is computed but not persisted; blocked tasks show their
  recorded reasons.
- The one-line coverage summary is present and reuses the T-059 coverage
  capability (no duplicated computation).
- Tests cover the read-only guarantee and the JSON shape.
- `README.md` and `CHANGELOG.md` are updated for the new command.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass.

## Verification Notes

- Run `status` and `status --json` in a sandbox; assert `git status` stays clean
  and the JSON matches the human view.
