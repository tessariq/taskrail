# T-015 Add release automation via GoReleaser

## Description

Automate cutting Taskrail releases so that pushing a `v*` tag produces published,
cross-platform binaries on a GitHub Release. Today tagging is a fully manual maintainer
action (documented in the T-014 release checklist under the gitignored `planning/artifacts/verify/` tree): build with
`-ldflags`, create the tag, push. This task wires a GoReleaser-based pipeline modeled on
the `voxclip` `release.yml`, trimmed to Taskrail's pure-Go, no-cgo, no-native-deps reality
(no whisper build job, no Homebrew tap unless explicitly wanted later).

This is release infrastructure only; it adds no product (CLI) behavior.

## Acceptance

- A `.goreleaser.yaml` builds `./cmd/taskrail` for `linux` and `darwin`, `amd64` and `arm64`, injecting the version via `-ldflags "-X main.version={{.Version}}"`.
- `taskrail version` from a GoReleaser-built binary reports the tag (e.g. `v0.1.0`), not `0.0.0-dev`.
- `.github/workflows/release.yml` triggers on `push` tags matching `v*` and on `workflow_dispatch` (snapshot, no publish).
- On a `v*` tag push, the workflow runs GoReleaser and publishes archives to a GitHub Release for that tag.
- Release notes are extracted from the matching `## v<version>` section of `CHANGELOG.md`, with a sane fallback to the tag name when no section is found.
- A pre-publish guard fails the release if the tag version has no corresponding `## v<version>` heading in `CHANGELOG.md` (Taskrail injects version via ldflags and has no in-code version constant, so the changelog heading is the source of truth to check against the tag — not a hardcoded `version.go`).
- `workflow_dispatch` runs GoReleaser in `--snapshot` mode and does not create a GitHub Release.
- `go vet ./...`, `gofmt -l .`, and the existing CI remain green; `taskrail validate` still reports a valid repo.

## Verification Notes

- Validate the GoReleaser config locally with `goreleaser check` and a dry `goreleaser release --snapshot --clean`, capturing output under the gitignored `planning/artifacts/verify/` tree.
- Confirm a snapshot-built binary's `taskrail version` matches the computed version.
- Do not push a real tag as part of verification; the maintainer cuts the actual `v0.1.0` release once this lands.
