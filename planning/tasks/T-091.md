# T-091 Harden taskrail:check Windows compatibility: exercise on windows-latest CI leg and surface freshcheck cleanup errors

## Description

Follow-up from T-082 (go-reviewer, two LOW findings). T-082 made the
`taskrail:check` freshness guard portable to native Windows by replacing
mktemp/cmp/trap with the cross-platform `internal/toolchain/cmd/freshcheck`
helper. Two residual gaps remain, both about hardening the Windows path rather
than fixing a live defect:

1. **No live Windows execution of the guard.** `task taskrail:check` runs only in
   the `checks` job on `ubuntu-latest`; the `windows-latest` runner lives in a
   separate `build` job that never calls it (`.github/workflows/ci.yml`). The
   portability claim is verified by code inspection, stdlib-only cross-compile,
   and the `TestTaskrailCheckIsPortable` string-guard, but not by an actual
   Windows run.
2. **`freshcheck` swallows its cleanup error.** `defer os.Remove(fresh)` in
   `internal/toolchain/cmd/freshcheck/main.go` discards the error, so a failed
   removal (e.g. a Windows file lock) leaves the throwaway
   `bin/taskrail-freshcheck(.exe)` behind with no diagnostic. Self-healing on the
   next build, but a silent swallow contrary to the handle-errors-explicitly
   guideline.

Spec: `specs/v0.3.0.md#goals` (build/CI hygiene, single-source toolchain).

## Acceptance

- CI exercises `task taskrail:check` on `windows-latest` (add the step to the
  existing Windows matrix leg after `task taskrail:install`), confirming a fresh
  build passes there; the POSIX/CI leg on `ubuntu-latest` still passes.
- `freshcheck` no longer silently discards its cleanup error: a failed
  `os.Remove` of the throwaway build warns to stderr without failing the check
  (removal failure must not flip a fresh binary to "stale").
- Byte-reproducible build invariant intact (`CGO_ENABLED=0 -trimpath`), guarded
  by `TestTaskrailBuildsShareReproducibleFlags`.
- `gofmt -l .`, `go vet ./...`, `go test ./...`, and `taskrail validate` pass;
  `internal/toolchain` guards still pass.

## Verification Notes

- Confirm the `windows-latest` CI leg runs `task taskrail:check` green against a
  fresh build; confirm `ubuntu-latest` still passes.
- Add/adjust a unit test for the cleanup-warning path in
  `internal/toolchain/cmd/freshcheck`.

## Implementation Notes

- Origin: T-082 go-reviewer review, two LOW findings
