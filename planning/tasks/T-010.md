# T-010 Implement task transitions

## Description

Implement `start`, `complete`, and `block` transitions that update both the task files and `planning/STATE.md`.

## Acceptance

- `start` marks one task active.
- `complete` marks a started task completed.
- `block` records a blocked task and reason.
- State remains consistent with task status.

## Verification Notes

- Add tests for valid and invalid transitions.
- Run CLI smoke checks for the transition commands.
