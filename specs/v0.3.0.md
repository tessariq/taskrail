# Taskrail v0.3.0

## Summary

Taskrail `v0.3.0` explores the strategic extension area around spec-task consistency, decomposition quality, and planning drift.

This is where Taskrail can become significantly more valuable than a deterministic task-state CLI.

## Goals

- improve alignment between intent and execution
- detect missing work earlier
- detect drift between specs and tasks
- help ensure planning remains grounded in the active spec

## Potential Features

### Spec-To-Task Decomposition Support

Help derive draft tasks from spec documents.

This may be:

- a guided workflow
- an import mode
- a prompt or skill-assisted flow
- later, optional built-in AI support

### Coverage Checks

Check whether spec requirements are represented by tasks.

Questions:

- Are there spec areas with no tasks?
- Are core requirements uncovered?

### Orphan Task Checks

Check whether tasks are not meaningfully grounded in current specs and goals.

Questions:

- Are there tasks with no spec linkage?
- Are there outdated tasks detached from current intent?

### Drift Detection

Check for inconsistencies between current specs and the current task graph.

Questions:

- Did the spec evolve without corresponding tasks?
- Did tasks evolve beyond or away from the spec?

### Gap Analysis

Help identify likely missing work such as:

- tests
- verification tasks
- docs
- migration
- rollout
- edge cases
- cleanup

## Caution

Do not turn this into:

- a generic PM system
- a hosted planning platform
- a giant agent framework

Keep it grounded in:

- repo-native files
- explicit structure
- inspectable outputs
- human reviewability
